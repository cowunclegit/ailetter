# Implementation Plan: Content Links and Visual Thumbnails

**Branch**: `008-content-links-thumbnails` | **Date**: 2026-01-13 | **Spec**: [specs/008-content-links-thumbnails/spec.md](spec.md)
**Input**: Feature specification from `/specs/008-content-links-thumbnails/spec.md`

## Summary
Enhance the trend collection system to include visual previews and direct links. For YouTube, we will extract high-resolution thumbnails via API. For RSS, we will scrape OpenGraph tags during the background collection job. The frontend and newsletter templates will be updated to display these thumbnails and make titles clickable.

## Technical Context

**Language/Version**: Node.js (Backend), React.js (Frontend)
**Primary Dependencies**: `metascraper`, `metascraper-image` (Scraping), `axios`, `googleapis` (Existing), `MUI` (Frontend)
**Storage**: SQLite3 (Updated `trend_items` schema)
**Testing**: Jest (Unit & Integration)
**Target Platform**: Web Browser / Email Clients
**Project Type**: Web application
**Performance Goals**: < 200ms increase in dashboard load time; Scraping occurs in background jobs.
**Constraints**: YouTube API quota limits; Possible missing OG tags in some RSS sources.
